String vs Int: The Battle for Palindrome Plains

Oct 31st, 2010

I have a homework for the University in which I have to say if a number is a palindrome or not. So I was wondering which method is going to be faster? Reading it as a string and checking it character by character or reading it as a int and comparing it to it’s reverse?  INT Pros: Faster checking Cons: Had to generate the reverse of it which included using the % (mod) operator which we all know is time consuming STRING Pros: Only had to check the first letters of a number in the case the number wasn’t palindrome. Cons: Had to use strlen to get the length of the string. So I had written some simple scripts pretty fast and I was amazed to see that on a random dataset the string method is somewhere between 10-20% faster. I made an archive with all the scripts and the random number generator, you can get it here. I am pretty sure that I can make a way in which I can find the length of the string from reading it, without reading character by character. What do you say?
